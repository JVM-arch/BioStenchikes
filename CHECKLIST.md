# ✅ Чеклист перед деплоем на GitHub Pages

## Перед первым деплоем

- [ ] Создан репозиторий на GitHub
- [ ] Код загружен в репозиторий
- [ ] В `vite.config.js` указан правильный base path (если репозиторий не `username.github.io`)
- [ ] Файл `.nojekyll` создан в папке `public/`
- [ ] GitHub Pages включен в Settings → Pages (Source: GitHub Actions)

## После деплоя

- [ ] Сайт открывается по правильному URL
- [ ] Основная страница загружается корректно
- [ ] Админ-панель доступна по `/#/admin/profile`
- [ ] Стили и шрифты загружаются правильно
- [ ] Изображения отображаются
- [ ] Роутинг работает (hash routing)

## Настройка API (опционально)

- [ ] Remove.bg API ключ добавлен в GitHub Secrets (если используется)
- [ ] Переменные окружения настроены в workflow

## Проверка функциональности

- [ ] Админ-панель открывается (Ctrl+Alt+A)
- [ ] Можно редактировать профиль
- [ ] Настройки сохраняются
- [ ] Загрузка файлов работает
- [ ] Удаление фона работает (если настроено)

## Troubleshooting

Если что-то не работает:

1. **404 ошибка**: Проверьте base path в `vite.config.js`
2. **Стили не загружаются**: Убедитесь, что `.nojekyll` файл создан
3. **Роутинг не работает**: Используйте hash routing (`#/admin/profile`)
4. **API не работает**: Добавьте секреты в GitHub Actions
